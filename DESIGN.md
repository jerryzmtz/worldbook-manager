# 世界书管理器设计规范

## Design Source

当前设计规范以 `src/世界书管理器/App.vue` 中的世界书缓存优化器为权威样板。新增或调整功能时，优先复用现有 `wbm-*` 类、CSS 变量、按钮形态、面板结构和文案密度。

## Visual Register

- 类型：深色产品工具界面。
- 使用场景：用户在 SillyTavern / TauriTavern 内维护已有数据，需要密集扫描、比较和确认。
- 气质：克制、清晰、可操作。避免营销感、装饰感和与宿主页面混杂。

## Layout

- 主弹窗使用居中 overlay，最大宽度和高度由 `.wbm-dialog` 控制。
- 顶部为 sticky header，左侧显示面板标题和版本，右侧放设置、教程、关闭等图标按钮。
- 配置区使用双列网格：
  - 左列为规则、模式、策略或说明区。
  - 右列为世界书选择列表。
  - 左列宽度约 `300px-360px`，右列占剩余空间。
- 移动端和窄屏改为单列：
  - 规则区在上。
  - 世界书列表在下。
  - 预览和操作区保持可滚动、可折叠。
- 预览区位于配置区下方，承载建议、风险、差异和应用按钮。

## Components

- 面板使用 `.wbm-panel`：深色表面、细边框、18px 左右的大圆角、12px 内边距。
- 按钮使用现有 `.wbm-primary-btn`、`.wbm-small-btn`、`.wbm-danger-btn`、`.wbm-icon-btn`。
- 图标使用 Font Awesome，优先用于设置、刷新、说明、关闭、风险提示和状态。
- 输入和选择控件使用 `.wbm-input`、`.wbm-select`，保持高度、边框、focus 样式一致。
- 世界书列表使用 `.wbm-book-list` 和 `.wbm-check-row`，来源标签使用 `.wbm-source-badge`。
- 风险和提示使用 `.wbm-alert`、`.wbm-dedupe-warning-line` 等已有语义组件。
- 设置弹窗使用现有 modal 结构和 radio choice list，不为单个功能发明新控件。

## Color And Tokens

- 所有新增样式优先使用 `--wbm-*` token。
- 主背景使用 `--wbm-bg`，面板使用 `--wbm-surface` / `--wbm-surface-raised`。
- 文本使用 `--wbm-text`，次要说明使用 `--wbm-muted`。
- 主要操作和选中状态使用 `--wbm-blue`、`--wbm-blue-soft`、`--wbm-blue-softer`。
- 风险、失败、警告沿用现有黄色、红色和绿色语义色，不新增不必要的高饱和色。

## Typography

- 使用系统 UI 字体栈，保持产品工具的原生感。
- 面板标题紧凑，正文和说明使用小字号但不低于现有可读标准。
- 按钮和标签文案短句优先，避免长段说明塞进控件。
- 提示文案要说明影响和风险，不重复标题。

## Interaction Rules

- 修改和删除类操作必须先生成方案，再由用户确认。
- 低置信、跨名称、激进策略、绑定重写等风险必须在预览中可见。
- 设置变更应即时保存到 localStorage，但不自动执行数据修改。
- 加载、读取失败、应用失败必须可见反馈。
- 新功能应保持与缓存优化器一致的配置区和预览区节奏。

## Anti Patterns

- 不把规则区放到右侧，除非整个工具重新设计。
- 不使用嵌套卡片堆叠规则说明。
- 不新增装饰性渐变、玻璃拟态或大面积单色主题。
- 不用不同按钮形态表达同一层级操作。
- 不把用户需要确认的风险藏在 tooltip 或日志里。
